# 「尋找調代課小幫手」維護說明

> 給接手維護的教學組長 / 行政同仁。
> 拿到新學期 PDF 後，照下面步驟做就能更新出新的 HTML 給全校用。

## 工具是做什麼的？

把全校老師的紙本／PDF 課表轉成一份**單一 HTML 檔**（約 580 KB），老師打開就能：

1. 選擇自己 → 看自己的課表
2. 點請假時段 → 看誰可以代／調
3. 一鍵複製訊息／截圖／分享給對方

整份 HTML **不需要伺服器**，可以放雲端硬碟、學校內網、隨身碟，雙擊開啟即可。

---

## 一、一次性的環境準備（第一次才需要）

### 1. 安裝 Python（建議 3.10 以上）

到 [python.org](https://www.python.org/downloads/) 下載安裝。**安裝時記得勾選 “Add Python to PATH”**。

### 2. 安裝唯一一個第三方套件

開啟 PowerShell（Windows）或 Terminal（Mac），執行：

```powershell
pip install pdfplumber
```

> 這套件用來解析 PDF 表格。其他都用 Python 內建模組。

### 3. 拉專案到本機

```powershell
git clone <專案 Git 連結>
```

或直接拷貝整個 `課表分析/` 資料夾。

### 4. 建立姓名筆劃對照表（只跑一次，之後永遠不用再跑）

```powershell
python scripts/build_strokes.py
```

這會下載 Unihan 資料庫（8 MB），生成 `school_wide/_strokes.json`。**之後無論換幾次學期都不用再跑**，除非檔案被刪了。

## 二、新學期更新步驟（重點，每學期跑一次）

假設你拿到一份新的全校課表 PDF。整個更新只要 **4 步**：

### 步驟 1️⃣：把新 PDF 放進 `source/` 資料夾

把新檔案命名為 `全校課表.pdf`，**取代**舊的同名檔。

```
課表分析/source/全校課表.pdf  ← 替換這個檔
```

> ⚠️ 注意：腳本只認這個固定檔名。如果你要保留舊檔，請改名為 `全校課表_113-2.pdf` 之類。

### 步驟 2️⃣：抽取 PDF → CSV

```powershell
python scripts/extract_school.py
```

成功後會看到類似：
```
[ok] school_wide/全校課表_長表.csv: 1816 筆 / 132 老師 / 132 班
     IB 教師：35 位 / 班級導師：48 位
```

產出三個檔（在 `school_wide/`）：
- **`全校課表_長表.csv`** — 1800 筆左右課程紀錄，**這是後面所有東西的真相來源**
- `全校_extraction_report.md` — 抽取過程報告（可疑欄位、缺漏統計）
- `全校_領域時間.md` — 「領域時間」獨立列出

### 步驟 3️⃣：產出分類確認表，**人工檢查**

```powershell
python scripts/audit_categories.py
```

會生出 [`school_wide/分類確認表.md`](school_wide/分類確認表.md)，**請打開來掃一眼**，特別注意這幾節：

| 章節 | 該檢查什麼 |
|------|------------|
| 1. 主授科目分布 | 各科人數合理嗎？（國/英/自/數/社/藝/體/特/二外/本土語） |
| 2. 每位老師主授科目 | 有沒有人科目被歸錯（特別是兼任老師） |
| 3. IB 教師（35 位） | 有沒有少 IB 老師、或誤把普通班老師當 IB |
| 4. 班級導師（48 位） | 對照學校導師名單是否吻合 |
| 5. IB 課程清單 | 純英文課名是否都列出來 |
| 8. 各主授科目對應的課程 | 自/國/英/數… 各科課程清單合不合理 |

**如果發現分類錯誤** → 回去調整 `scripts/extract_school.py` 內的分類規則（找 `classify_course_subject`、`classify_course_detail`、或 `PREFIX_TO_SUBJECT` 等函式）→ 重跑步驟 2 + 3。

> 💡 小撇步：日常微調（譬如「某老師主授要從『藝』改成『體』」）通常改 `PREFIX_TO_SUBJECT` 的代號前綴對應表就能搞定。

### 步驟 4️⃣：生成最終 HTML

```powershell
python scripts/build_web_school.py
```

成功後會看到：
```
[ok] 已寫出 school_wide/代課查詢_全校.html
     資料：1816 筆課程 / 130 位老師
```

這個檔就是給老師用的成品。

### 步驟 5️⃣：更新頁面上的「資料版本」標示

打開 [`scripts/build_web_school.py`](scripts/build_web_school.py)，找到約在 `<h1>` 那行：

```python
<h1>尋找調代課小幫手 <span ...>資料版本 114-2 · chaher@dysh.tyc.edu.tw × Claude Code</span></h1>
```

把 `114-2` 改成本學期（例如 `115-1`），改完再跑一次步驟 4 即可。

## 三、更省力做法：用 VSCode + Claude Code 對話完成

如果你不想記指令、不想開 Terminal，可以用 **VSCode** 加上 **Claude Code 擴充套件**直接用中文「講」給 Claude 聽，它會自己跑指令、改檔案、回報結果。下面這套是這份說明預設的工作流程。

### 1. 一次性安裝（首次才需要）

1. 安裝 [Visual Studio Code](https://code.visualstudio.com/)（免費）
2. VSCode 左側 Extensions（四方塊圖示）搜尋 **Claude Code**，點 Install
3. 點側邊出現的 Claude 圖示 → 登入 Anthropic / Claude 帳號

> 學校若沒有公帳號，可在 [claude.ai](https://claude.ai) 自己申請。

### 2. 打開專案

VSCode → File → Open Folder… → 選 `課表分析/` 整個資料夾。

### 3. 把新 PDF 放進去後，直接跟 Claude 講

新學期 PDF 拖進 `source/`，命名為 `全校課表.pdf` 取代舊檔。然後在 Claude Code 對話框打：

```
新學期的全校課表 PDF 已經放好了，幫我跑完整流程：
抽取、產分類確認表、生成最後的 HTML，並把資料版本改成 115-1
```

Claude 會：
- 依序執行步驟 2 → 3 → 4 → 5 的所有指令
- 順便修改 `build_web_school.py` 裡的版本字樣
- 跑完回報「IB 教師 35 位、班級導師 48 位」這類數字，提醒你最值得人工確認的對象

### 4. 各種特殊狀況的範例對話

| 想做的事 | 直接打給 Claude |
|----------|------------------|
| 改某位老師的分類 | 「林家禾 這位老師主授被分成 X，幫我改成 Y」 |
| 換署名 email | 「把 HTML 上的署名 email 換成 abc@dysh.tyc.edu.tw」 |
| 改網頁標題、文案 | 「請把 XX 文字改成 YY」 |
| 看分類有沒有問題 | 「幫我列出最值得人工確認的老師有哪些、為什麼」 |
| 出錯訊息看不懂 | 把終端機紅字貼給它，問「這是什麼意思？」 |
| 想知道某個按鈕 / 顏色怎麼改 | 「按確認代課的綠色想換成藍色，幫我改」 |

### 5. 安全使用提醒

- Claude 改任何檔案**會先預覽 diff 給你看**，看不懂或不確定就按「拒絕」
- 真正給老師發布的檔案永遠都是 `school_wide/代課查詢_全校.html`，其他都只是中間檔
- 不確定 Claude 剛剛做了什麼 → 直接問它「列出剛剛改了哪些檔案、改了哪幾行」
- **修改前先用 git 確認你的工作有 commit**：這樣即使改壞了也能還原

## 五、發布給全校的方式

最終 `school_wide/代課查詢_全校.html` 是**單一檔案**（約 580 KB，已內嵌所有 JS 函式庫），任何傳檔方式都可以：

1. **學校 Google Drive 共用** — 設定「校內所有人可檢視」最方便
2. **學校官網／教務處頁面內嵌**
3. **Email 附件群發**
4. **印 QR code 給老師掃** — 手機也能順暢開啟

> ⚠️ 老師資料屬於內部資料。如果上傳到公開網路（例如 GitHub Pages），請先確認學校對課表公開的政策。

## 六、常見問題

### Q1: `pip install pdfplumber` 失敗怎麼辦？

通常是公司／學校網路擋外部 PyPI。試：
```powershell
pip install pdfplumber -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q2: 跑 `extract_school.py` 出現「`source/全校課表.pdf` not found」

確認 PDF 檔名**完全是** `全校課表.pdf`（注意全形/半形、底線、空白等）。Windows 檔案總管預設隱藏副檔名 → 在「檢視」裡打開「副檔名」確認。

### Q3: HTML 打開後選單空空的？

99% 是 CSV 裡沒資料。回頭看步驟 2 的輸出有沒有「1816 筆課程」這行；若數字異常少，多半是 PDF 格式變了，要請工程支援。

### Q4: 某位老師被分類錯了

先看 `分類確認表.md` 的第 2 節確認誰錯。常見修法：
- **代號開頭族群錯**（例如 A 開頭老師被歸成「自」）→ 改 `extract_school.py` 的 `PREFIX_TO_SUBJECT`
- **個別老師例外**（例如某 F 老師被歸到體育）→ 在 `extract_school.py` 加個別覆蓋規則
- **IB / 普通班分錯** → 通常代表那位老師的 PDF 有純英文／中文課程被誤判，回去看他的課程清單

改完規則後重跑步驟 2、3、4。

### Q5: 「資料版本」要怎麼改？

打開 `scripts/build_web_school.py`，找到（約在 `<h1>` 那行）：
```python
<h1>尋找調代課小幫手 <span ...>資料版本 114-2 · ...</span></h1>
```
直接改文字後重跑 `build_web_school.py`。

### Q6: 想把署名換成自己的 email？

同上，在 `build_web_school.py` 同一行：
```python
資料版本 114-2 · chaher@dysh.tyc.edu.tw × Claude Code
```
直接改 email 即可。

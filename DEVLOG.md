# Dev Log — 全校代課查詢網頁

時間軸與設計決策記錄。供其他工程師了解需求演進與我們做出的取捨。

## 起點

自然科 16 位老師的代課查詢網頁（`代課查詢.html`）已經穩定運作。資料源是手動校對過的 `課表_長表.csv`（229 筆），UI 將候選代課老師依「主授科目」（物/化/生/地科）分組顯示。

## 階段一：擴大到全校 130 位老師

### 需求

擴大到全校 130 位老師。先從 PDF 抽出 1816 筆課程資料到 `全校課表_長表.csv`，需要新增「教師類別」欄位區分 IB 教師與普通班教師。

### 第一版規則：含英文字母 = IB

> 找出課程名稱有用英文寫的，這些教師定義為 IB 教師，其餘普通班。

直觀但**誤判嚴重**：
- `英語文 B` 的 B 是普通班分組標記 → 14 位英文老師被錯標為 IB
- `數學 A` / `數學 B` → 5 位數學老師
- `日語 A` / `日語 B` → 2 位
- `Fun 影片高一多元`、`EWANT 數位學習` → 3 位 MOOC/多元課老師
- 其他 CSV 解析瑕疵造成的偽英文 → 3 位

### 第二版：嚴格 IB 字串 + 教室證據

改用 IB 慣例字串列表精準匹配：
```
SL/HL/TOK/CAS/IPSS/ChineseA/EnglishA/EnglishB/Mandarin/VA-/MathAA/MathAI/ChiBS
```

對於課名沒明確 IB 標記但以教室代碼佐證為 IB 的 10 位老師（在 IB04~IB14、114A-116A 教室授課），手動列入 IB 教師。

最終 35 位 IB 教師、95 位普通班教師。

## 階段二：發現 PDF 解析瑕疵

校對時發現 12 筆奇怪的列：
```
A18,楊名傑,2,5,圖書館自高二自主S1304(Se,,
C10,余昕嬪,2,5,高二自主高二自主S1303(Li,,
```

教室代碼 `S1xxx(...)` 被吞進課程名稱欄位。追到根因：`extract_v2.py` 的 `is_room_like()` 只認 `T/A/IB` 開頭的教室碼，沒處理 `S` 開頭。

S 開頭教室都是「自主學習教室／圖書館」性質，剛好就是 自主/充實/多元 這類沒指定班級的彈性課程在用。

修正：
1. 改 `is_room_like()` 加 `S` 開頭判斷（未來重抽不會再出問題）
2. 一次性修正現有 12 筆 CSV 列（拆出教室碼、補上右括號）

## 階段四：建構全校代課查詢 v1

新增 `build_web_school.py`，沿用自然科版的 UI/JS 結構，但：
- 候選分組改用 `教師類別`（IB / 普通班）
- 130 人選單加上 `<datalist>` 搜尋輔助

跑出來的網頁能用，但**設計有問題**：UI 上掛著「IB 教師」「普通班教師」標籤。

## 階段五：UX 大改 — 去階級化、改用主授科目分組

### 使用者回饋（關鍵）

> 普通班教師、IB教師，這樣的分類總是不會出現在介面上（很多老師會誤會有階級之分）

階級觀感是真實顧慮。即使內部資料模型分 IB/普通，UI 不能露出。

### 改設計

| 項目 | 改前 | 改後 |
|------|------|------|
| 候選主分組 | 同類別 / 其他類別 | 同科 / 其他科 |
| 「其他科」拆細 | 無 | 拆 國/英/自/數/社/其他 |
| 老師標籤 | `（IB教師）` | `（科目）` |
| IB 課程例外 | 無 | IB同科 → IB其他科 → 同科 → 其他科 四階優先 |
| 教師代號搜尋 | 無 | datalist 同時收姓名與代號 |
| 同組內排序 | 無特定 | 教師代號由小到大 |

### 衍生的資料工程問題

「主授科目」CSV 沒這欄，要從每位老師的課程自動推。寫了 `classify_course_subject()` 規則：
- 特殊 IB 先判斷（TOKCh→國、TOKEng→英）
- 社會優先（避免 `歷史學探究` 被「探究」抓走歸到自）
- 篩掉彈性課（行政/團體/自主/多元/充實/深廣/領域時間/Adm sup/Eng sup/CT/TOK/CAS/INS）
- 再依字串特徵分國/英/數/自/社
- 沒命中 → 其他

每位老師主授 = 計入課程中出現次數最高的類別。少數曖昧案例（馬汝強 5 自 vs 5 數打平）就接受演算法的選擇，使用者可以單筆改 CSV 覆蓋。

### IB 課程偵測

JS 端用 `IB_PATTERNS` 字串陣列做 substring 比對，跟 CSV 教師類別不重複。決定該節要不要走「IB 4 階優先順序」是看那節的課名，不是看老師是否為 IB。

## 階段六：工作流程定稿

CSV 是**單一真相來源**，HTML 永遠由 build script 重生。但要小心：

- 改 CSV → 跑 `build_web_school.py`（安全）
- 改 CSV → 跑 `extract_school.py`（**手動修正會被覆蓋**）

「資料型」修改（單筆教師主授錯）→ 改 CSV。
「規則型」修改（要把 BMSL 從社改到自）→ 改 `extract_school.py` 的分類器，重抽。

## 階段七：同班 / 導師優先

### 需求

> 預設候選順序：同科 → 同班級 → 其他科。同班任課老師中，導師排最前面（提醒使用者：調課優先找導師）。
> IB 課程：IB同科 → IB同班 → IB其他科 → 同科 → 其他科。

### 實作

- `extract_school.py` 新增 `extract_homeroom_class()`，從 PDF header 抽 `(\d{3}[A-Z]?)\s*導師` 模式 → 48 位班級導師偵測成功
- F09 張雲靜（特教導師）無班號，正確回空字串
- CSV 加 `導師班級` 欄
- `build_web_school.py`:
  - 預先建立 `KLASS_TO_TEACHERS` 索引（O(N) once）以判斷「同班候選」
  - 候選邏輯改 3 階／5 階互斥分群
  - `sortCandidates()` 一律把該班導師排第一
  - 候選按鈕若是該班導師，姓名後加紅色「（導師）」字樣

### 為什麼導師標記要紅字

使用者反映：**調課實際慣例就是優先找導師**。要讓導師身分一眼可辨，紅字 + 「（導師）」是教育界既定的視覺語言。

## 階段八：科目細分與分類審查表

### 需求

> 科目「其他」太籠統，應再細分：音樂/美術/藝術/生科/資訊 → 藝；體育 → 體；日/德/法/西 → 二外。
> 我們有許多歸類（IB 老師、IB 課程、領域時間、自主課程、導師…），輸出一份 md 讓我確認。

### 實作

`classify_course_subject()` 從 6 類擴充為 9 類：
- **藝**：音樂、美術、藝術生活、新媒體藝、生科、資訊科技、VA/PA/AW（7 位）
- **體**：體育、PE、健護、運動、專項技術／體能、SEHSSL（13 位）
- **二外**：日語、德語、法語、西班、西語、韓語（9 位）

「二外」檢查移到 flex 之前，避免 `韓語高一多元` 這類僅有彈性課的純語言老師被吞到「其他」（修了 L23 陳湘云）。

### 分類審查表

新增 `scripts/audit_categories.py` → `school_wide/分類確認表.md`，內容含：
1. 主授科目分布（9 類）
2. 每位老師主授／IB／導師班一覽
3. IB 教師清單（35 位）
4. 班級導師清單（48 位，依班號排）
5. IB 課程清單（matching 13 個 IB pattern）
6. 含「領域」課程（區分領域時間 vs 領域課程）
7. 自主／自學相關課程
8. **各主授科目對應的實際課程列表**（讓人一眼看出分類器把哪些課歸到哪一類）

### 9 類最終分布

| 國 | 英 | 數 | 自 | 社 | 藝 | 體 | 二外 | 其他 |

## 階段九：代號 prefix 強制覆蓋

### 需求

> 原則上同代號就會是同科目族群 → 全部都強制用代號歸類。

驗證後確認 6 個例外都是 IB 跨領域老師（如 E41 Kennedy 教 BMHL/BMSL 但代號 E）或單一例外（F06、G22）。學校內部的代號 = 行政組織分類，比實際授課科目更貼近「找哪位同事代課」的真實邏輯。

### 實作

`extract_school.py` 在 course-based 分類算完後，疊一層 prefix override：

```python
PREFIX_TO_SUBJECT = {
    "C": "國", "E": "英", "F": "社", "G": "體",
    "L": "特", "M": "數", "N": "自", "S": "社",
}
# A 例外：沿用課程式分類（A 內部真的混合 體/藝/家政/資訊/生科）
```

### 結果

| C 17 | E 20 | F 5 | G 1 | L 15 | M 16 | N 16 | S 17 |
|------|------|-----|-----|------|------|------|------|
| 國 | 英 | 社 | 體 | 特 | 數 | 自 | 社 |

A 族群維持課程分類：體 14 / 藝 8 / 社 1（A15 IB Psy 老師）。

「二外」與「其他」類別在覆蓋後實際上沒有教師（二外 9 位都歸 L→特；其他 3 位都被 prefix 收編）— 在 UI 中仍保留為候選類別位置，留給未來可能的單筆覆蓋。

### 改動 vs course-based 的差異（10 位老師被勘正）

| 老師 | 原 (course) | 新 (prefix) |
|---|---|---|
| E26 Charlie | 其他 | 英 |
| E41 Kennedy | 社 | 英 |
| F06 蔡亞彤 | 其他 | 社 |
| F07 林郁婕 | 其他 | 社 |
| G22 徐敏凱 | 其他 | 體 |
| M36 馬汝強 | 自 | 數 |
| L (15 位) | 二外/國/其他 | 特 |

對代課用途：找代課時優先看的是「這位老師是哪個辦公室／組別的同事」，而非「他真實授課科目是什麼」。學校的代號制度本身就是組織歸屬，prefix override 直接對齊。

## 階段十：細科目子排序

### 需求

> 有時需要找特定細科（譬如物理化學都是自然，但有時要找物理）。社會科同理。

### 實作

CSV 加 `細科目` 欄。從每位老師的課程推：物/化/生/地科、史/地理/公民/法律/心理/商管、音樂/美術/家政/視覺藝術/表演藝術/資訊/生科、體育/健護/安全教育/全民國防、日/德/法/西/韓/客語/閩南語/原住民語、國文/英文/數學…約 30 種。

UI 在「同科」群組內子排序：**該班導師 → 同細科 → 其他（按代號）**。
- 候選人按鈕加 `★` 綠字標記同細科。
- 括弧內顯示「主授／細科目」（細科與主授相同則只顯示主授，避免冗餘）。

### 結果（自然科 N 族群）

| N01 | N02 | N03 | N04 | N06 | N11 | N38 | N45 | N49 | N51 | N52 | N59 | N60 | N63 | N68 | N69 |
|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|
| 物 | 化 | 生 | 地科 | 化 | 生 | 物 | 化 | 物 | 生 | 地科 | 生 | 物 | 物 | 化 | 化 |

物理老師請假找代課時，同樣是物理的 N01/N38/N49/N60/N63 會在 N 族群內排在最前面，化/生/地科老師排後面。

### Trade-off

馬汝強（M36）IB 雙主修 物理+數學，主授=數（prefix override），細科=物（按課程主流）。在 N 族群（物理）的 leave slot 中**不會**被列為候選 — 因為主授已固定在 M。屬於 prefix override 設計的可預期取捨。

## 階段十一：姓名筆劃排序

### 需求

> 群組內排序由「教師代號」改成「姓名筆劃 小→大」。對使用者來說，找老師看姓名比看代號直覺。

### 取捨：套件選擇

兩個方案：
- A. 用 Python 套件算筆劃，build 時生成已排序資料 → HTML 端零變動
- B. JS 端 `localeCompare`／Unicode 點位排序 → 不裝套件，但**不是真的筆劃序**（部首排序與筆劃排序差很多）

選 A。但 `cnchar` 是 JS 套件，Python 沒對應。試了 `hanzipy`（recursive 部首拆解，不對）、`cjklib2`（不存在）、`unihan-etl`（只是下載工具）。

最後直接從 **Unicode Unihan 資料庫**（https://www.unicode.org/Public/UCD/latest/ucd/Unihan.zip）抽 `kTotalStrokes` 欄位。寫 `scripts/build_strokes.py` 下載 + 解析 + 快取，產出 `school_wide/_strokes.json`。230 個漢字全部抽到。

### 實作

`build_web_school.py` 載入 `_strokes.json`，計算每位老師全名筆劃，寫進 `TEACHERS[i].strokes`。
`sortCandidates()` 排序鍵改成：

```
該班導師 → 同細科 → 姓名筆劃 (小→大) → 教師代號 (deterministic tiebreak)
```

外籍老師（Andrew/Charlie/Kennedy）給 999 筆劃 → 排到該群組最後。

### 一個小瑕疵

陳 字 Unihan kTotalStrokes = 10（簡寫 convention），繁體字典算 11。差 1 筆不影響整體排序順位。如果要嚴格繁體字典筆劃，再寫個 override dict。

### N 族群按筆劃排序結果

```
黃可仁(21) → 陳以青(22) → 林家禾(23) → 周宥成(23) → 林邵洋(24) → 邱志強(25)
→ 蔡玉芳(26) → 蔡玉珍(28) → 李玉龍(28) → 詹子瑩(31) → 陳亭惠(31)
→ 周宸綺(32) → 張智豪(37) → 游誼萍(38) → 簡睿涵(43) → 趙興儒(46)
```

物理老師找代課（細科=物優先）→ 結果順序：林家禾 → 林邵洋 → 蔡玉珍 → 陳亭惠 → 黃可仁。

## 重要的設計取捨

1. **資料嵌入 HTML 而非 fetch CSV**：要讓 HTML 雙擊離線開（`file://` 下 fetch 被 CORS 擋）
2. **自然科與全校兩套 build pipeline**：自然科有人工校對的 gold standard 與專屬視圖；全校以 pdfplumber 直接輸出，工具集較精簡
3. **CSV 含三個衍生欄位（主授科目／教師類別／導師班級）**：denormalized 但每筆查詢不需要重算；資料嚴格由 `extract_school.py` 重生時保持一致
4. **UI 隱藏 IB 標籤**：避免階級觀感；內部資料模型保留以支援 IB 課程的優先順序例外
5. **同班候選只在非同科時顯示**：避免重複（同科同班會在 `同科` 群組中以「（導師）」標記呈現）
